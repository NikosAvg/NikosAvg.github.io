Dice Simulation in Python

Theory

Dice are used to provide random numbers for gambling and other games, and hence are a type of hardware random number generator. The result of a die roll is random in the sense of lacking predictability, not lacking cause. In a fair dice, each outcome (side) has the same probability of 1/6.

The implementation is easy; all we have to do is create a Python function that returns a random integer between 1 and 6. To check if the probabilities of all sides are equal, we can repeat this process enough times to have a sufficiently large sample size to perform some basic statistics.

Code

First, we need to import the necessary libraries to generate our random numbers and visualize the results.

import numpy as np
import matplotlib.pyplot as plt
plt.style.use('classic')

Then we create a function called diceRoll which generates a random integer between 1 and 6 and returns it to us.

def diceRoll():
    return np.random.randint(1, 7)

Next, we initialize a variable that stores the number of desired rolls, run our experiment, and store the results in a list.

number_of_rolls = 10000
results = []
for i in range(number_of_rolls):
    results.append(diceRoll())

We can now visualize our results by plotting them using a histogram.

plt.hist(results, density=True, bins=6)
plt.xlabel("Possible Outcomes")
plt.ylabel("Probability")
plt.title("Probability of Dice Throw Results")
plt.show()

We choose the number of bins to be 6 because these are our only possible outcomes, and we want to see how often each one appears.

The Sum of Two Dice

After seeing the results of a single dice throw, it’s only natural to consider the probability distribution of the sum of two dice.

Let’s first examine some of the theory of rolling two dice.

Theory

All possible combinations of rolling two dice, our sample space, are shown below:

1 2 3 4 5 6
1 (1,1) (1,2) (1,3) (1,4) (1,5) (1,6)
2 (2,1) (2,2) (2,3) (2,4) (2,5) (2,6)
3 (3,1) (3,2) (3,3) (3,4) (3,5) (3,6)
4 (4,1) (4,2) (4,3) (4,4) (4,5) (4,6)
5 (5,1) (5,2) (5,3) (5,4) (5,5) (5,6)
6 (6,1) (6,2) (6,3) (6,4) (6,5) (6,6)

As we can observe, there are 36 possible combinations of rolls, and each die acts independently of the other. Therefore, to calculate the probability of each pair, we need to multiply the probabilities of the individual outcomes. Since each individual outcome has a probability of 1/6, each pair of outcomes has a probability of 1/36.

Now we can visualize our results using another histogram.

plt.hist(results,density=True,bins = 11)
plt.xlabel("Possible Outcomes")
plt.ylabel("Probability")
plt.title("Probability of sum of two dice throw results")
plt.show()

We choose the number of bins to be 11 to represent the 11 possible outcomes of the sum of two dice throw, which range from 2 to 12.

As we can see from the histogram, the probability distribution of the sum of two dice throw follows a bell curve, with the most probable outcome being 7, which is the average of all possible outcomes.

In conclusion, we have seen how to simulate a dice throw in Python and how to use basic statistics to check if the probabilities of all sides are equal. We have also explored the probability distribution of the sum of two dice throw, which follows a bell curve. This technique can be extended to simulate more complex random events and can be useful in various fields such as finance, gaming, and scientific research.
